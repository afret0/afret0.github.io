redis note

SDS

常数复杂度获取字符串长度

减少修改 string 时的内存重新分配

增长操作如拼接字符串 (append) 等操作,需要重新分配内存来扩展底层数组的空间大小, 否则会缓冲区溢出

缩短操作如截断字符串 (trim) 等操作, 需要内重新分配来释放字符串不再使用的那部分空间, 否则会内存泄露

空间预分配

如果 SDS 修改之后, 长度(sds.len<1MB), 则分配 len 大小的未使用空间, 此时 sds.len == sds.free.

即修改过后, sds.len == 13byte, 则为 sds 分配 13byte 的未使用空间. sds 的 buf 数组实际长度则变成 13+13+1=27byte, 额外1
byte用于保存空字符串

如果 SDS 修改之后, sds.len >= 1MB, 则分配 1MB 未使用空间.

即修改过后, sds.len == 30MB, 那么会为 sds 分配 1MB 的未使用空间, sds 的 buf 数组实际长度则变成 30MB+1MB+1byte

通过这种预分配策略, SDS 重新分配次数从必定为 N 降低为最多 N 次.

惰性空间释放

当 SDS 需要缩短时, 并不会立即重新分配内存来回收, 而是使用 sds.free 属性记录.

链表

字典

Redis 的字典使用哈希表作为底层实现,一个哈希表可以有多个哈希表节点,每个哈希表节点保存字典中的一个键值对

哈希表

哈希表节点

字典

ht 属性是一个包含两个项的数组, 数组中的每个都是一个 dictht 哈希表, 一般情况下, 字典只使用 ht[0] 哈希表, ht[1] 哈希表只会在对 ht[0] 进行 rehash 时使用.

rehashindx 记录了 rehash 目前的进度, 如果目前没有 rehash, 则它的值为 -1.

哈希算法

当把一个新的键值对添加到字典的时候, 需要先根据键计算出哈希值和索引值, 然后根据索引值, 将包含新键值对的哈希表节点放在哈希表数组的指定索引上.

使用字典设置的哈希函数, 计算 key 的哈希值

hash = dict -> type -> hashFunction(key);

使用哈希表的 sizemask 属性和哈希值, 计算出索引值

index = hash & dict -> ht[x].sizemask;

解决键冲突

Redis 哈希表使用链地址法来解决键冲突, 每个哈希表节点都有一个 next 指针, 多个哈希表节点可以用 next 指针构成一个单向链表,被分配到同一个索引的多个节点可以用这个单向链表连接起来.

跳跃表 skiplist

跳跃表是一种有序数据结构,它通过在每个节点中维持多个指向其它节点的指针, 从而达到快速访问节点的目的

跳跃表支持平均 O(logN), 最坏 O(N) 的复杂度节点查找, 还可以通过顺序性操作来批量处理节点

Redis 使用跳跃表作为有序集合键的底层实现之一, 如果一个有序集合包含的元素数量比较多, 又或者有序集合元素成员是比较长的字符串时, redis 使用跳跃表来作为有序集合键的底层实现

Redis 只在有序集合, 和集群节点中用作内部数据结构.

跳跃表

skiplist 是一种有序结构, 通过在各个节点维持多个指向其它节点的指针, 来实现快速访问节点的目的.

skilplit 支持平均 O(logN),最坏 O(N)的节点查找, 还可以通过顺序性操作来批量处理节点.

整数集合

intset 是集合键的底层实现之一,当一个集合只包含整数值元素,并且集合元素数量不多时, Redis 就会使用整数集合作为集合键的底层实现.

添加新元素

要将一个新元素添加到整数集合里面,并且新元素的类型比整数集合现有元素类型都要长的时候, 整数集合需要先进行 upgrade 操作, 然后才能添加

升级并添加新元素需要三个步骤:

  1. 根据新元素的类型, 扩展 intset.contents 的空间, 并为新元素分配空间.
  2. 将 intset.contents 现有的元素换成与新元素相同的类型, 并将类型转换后的元素放到正确的位置上, 并且在放置的时候保持顺序不变.
  3. 将新元素添加到 intset.contents

intset 不支持降级操作

压缩列表

ziplist 是 LIST 和 HSET 的底层实现之一.

当一个列表键只包含少量列表项, 并且每个列表项要么是比较小的整数值, 要么是长度较短的字符串, Redis 则使用 ziplist 来作为列表键的实现.

对象

当我们对一个数据库键执行 TYPE 命令时, 返回的为对应的值对象的类型

ptr 指向对象的底层实现数据结构,由 encoding 属性决定.

使用 OBJECT ENOCODING 命令可以查看一个数据库键的值对象的编码.

字符串

字符串的编码可以是 int, raw, embstr

若保存的是整数值, 则可用 long 来表示, 字符串对象会将整数值保存在 prt 内.

embstr 是用来保存短字符串的一种优化编码方式.

如果保存的是一个字符串值, 且这个值的长度大于 32 字节, 则使用 SDS 来保存这个值, 并且编码设置为 raw

列表对象

列表对象的编码可以是 ziplist 或者 linkedlist.

当列表保存的字符串元素的长度都小于64字节, 且保存的元素的数量小于512, 则使用ziplist. 限值可通过配置文件修改.

哈希对象

哈希对象的编码可以使 ziplist 和 hashtabel.

ziplist 编码的哈希对象使用压缩列表作为底层实现,每当有新的键值对要加入到哈希对象时, 程序会将保存了键的 ziplist 节点推入到表尾,再将值推入到表尾, 因此, 保存了同一键值对的两个节点总是紧挨在一起, 键在前, 值在后.

当哈希对象保存的所有键值对的键和值的字符串长度都小于64字节, 且键值对数量小于 512个时, 使用 ziplist 编码.

集合对象

集合的编码可以是 intset 和 hashtable.

当集合保存的所有元素都是证书, 且集合数量不超过512个 则使用 intset 编码.

有序集合

有序集合编码使用 ziplist 和 skiplist.

有序集合元素量少于128, 且保存的所有元素成员都小于 64 字节时, 使用 ziplist.

内存回收

在创建一个新的对象时, 引用计数初始化为1

被新程序使用时, 引用计数+1

不再被使用时, 引用计数-1

当引用计数为 0 时, 内存会被释放.

空转时长

redisObject 含有 lru 属性, 记录了最后一次被程序访问的时间.

OBJECT IDLETIME 可以打印出给定建的空转时长, 通过使用当前时间减去 lru 计算出来的.

IDLETTIME 不会修改对象的 lru 值.
